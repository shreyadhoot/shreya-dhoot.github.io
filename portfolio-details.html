Project information

  • Title: Project Nexus
  • Project date: 01 Nov, 2023 - Present
  • Role: Project Lead
  • Technologies Used: C#, Next.js, React.js, RTK, Material UI, .Net Core, Microsoft Azure, Git, Figma, Microservices, CI/CD

Project Overview

The project aimed to simplify the Citrix Cloud Platform and make it more flexible to easily support multiple cloud accounts, environments, and geo-specific planes per Org ID.

Citrix Cloud Platform only allowed one environment or account per Org ID which meant customers had to create and manage multiple orgs and accounts to maintain different cloud environments like dev, staging, test, and production. The limitation was also on the cross-geo accounts, resulting in a very complex process to allow customers to allocate and distribute their licenses as needed.

Challenge

Bringing together the existing landscape under which customers had created multuiple organizations, under a single organization with multiple accounts was a major challenge. Setting the constraints on the number of tenants on organization level required a new process to be designed to evaluate the eligibility.

Achieved Outcomes

  1. With Nexus, we provided the flexibility to create multiple accounts including cross-geo under a single organization. The portal allowed the user to create and manage all the accounts at one place, along with allocation/deallocation and distribution of licenses within environments and across control planes.
  2. Removed all of the manual process involved in creating a new tenant, split and transfer of licenses across accounts. This brought down the process taking couple of weeks to a few seconds.
  3. Drastically reduced and cleaned up organization data to store and manage.Significantly reduced the cost of data stores.

Technical Achievements

  1. Successfully developed and timely delivered the first Next.js application keeping track of best practices and security.
  2. Laid down the Next.js application structure and guidelines to be followed for future JavaScript projects.

  • Role: Senior Software Engineer
  • Technologies Used: C#, Vue.js, MS SQL, Web API, MS Azure, CI/CD

Project Overview

The purpose of LA-007 was to reduce the number of production issues coming to developers by smartly identifying the issues across multiple systems and enabling the customer support to fix it with just a click.

After bringing the previously outsourced production support, in-house, there was an overwhelming increase in the workload on the developers that led to spending most of their time fixing production issues rather than the development deliverables.
The frustration gave me the idea to create a smart tool that would identify the recurring issues with customer license and fix it.

What started as a self-led project, soon gauge interest of leadership and the stakeholders, and became a full-fledged work project.
Later, looking at the results, it was added to the chatbot that allowed customer to fix it by themselves, removing the need to create a ticket itself.

Challenge

The number of systems involved in the license provisioning process and the categories of licenses available like - Hardware, Software, Cloud, Subscription, Perpetual, Termed, etc. increased the scope of errors and required to review all the systems and checks to identify the issues with a license. The manual process to debug or identify the issue easily took 4-5 hours.

Achieved Outcomes

  1. With LA-007, brought down the production support issues coming to our queue by >70%
  2. Enabled Customer Support and customers to fix the issues by themselves within a couple of seconds, which earlier used to take 1-2 weeks to get resolved.
  3. Proactively identified and fixed the customer issues.

Technical Achievements

  1. Successfully integrated the first tool with the chatbot to allow self-service.

  • Title: License Split & Transfer
  • Project date: 01 March, 2023
  • Role: Senior Software Developer
  • Technologies Used: C#, .Net Core, Microservices, MS Azure, MS SQL, Web APIs, CI/CD, Azure Service Bus

Project Overview

The purpose of this project was to enable the customers to perform split, partial and full transfer of their licenses between multiple organizations.

Challenge

Having the varied categories of licenses available like - Hardware, Software, Cloud, Subscription, Perpetual, Termed, etc. came with different configurations and product packaging. For licenses, that did not allow partial allocations required licenses to be split in the quantities as needed for each environment with an organization, and transfer to the other organization if needed. In case of acquisitions, the dormant organization required to transfer all the licenses to the now parent organization.


With subscription licenses, hardware and software were coupled together which required a new process to be designed so as to keep hardware intact and only split the software licenses.

Achieved Outcomes

  2. Enabled Customer Support and customers to fix the issues by themselves within a couple of seconds, which earlier used to take 1-2 weeks to get resolved.
